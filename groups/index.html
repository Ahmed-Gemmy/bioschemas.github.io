Join a group

To join Bioschemas please provide us with a Google account, a GitHub username and let us know which groups you would like to join via the Contributors spreadsheet. We will subscribe you to group mailing lists, invite you to the GitHub Bioschemas organization and give you access to the shared folders.

Create a new group

If you are interested in proposing a new group please create a new issue in GitHub with information about the group proposal including goal, purpose, expected products (i.e. specification), and people or projects you would like to involve. New group proposals will be brought to the community meeting organized once every month. In the community meeting we can review the information provided and agree on creating new groups.

Communication Channels

Regular monthly meetings

We recommend each Bioschemas group proposes regular meetings taking into account the availability of group members. Ideally the chair(s) of each group should announce meetings dates and make the minutes available in the Meetings wiki page.

Issue Tracker

We use the GitHub issue tracker to discuss ideas, issues and tasks related to Bioschemas.

Mailing Lists

We try to maintain one mailing list per group and a general mailing list (all@bioschemas.org), to which we subscribe everyone participating in Bioschemas. Mailing lists are open and accessible via Google Groups. We try to use the mailing list to send group members relevant information, but we prefer to keep discussions of ideas, issues and tasks in the GitHub issue tracker.

Shared Folders

We use Google Drive to share presentations, collaborative edit specifications and store any relevant file for our groups.

GitHub

We use the bioschemas.github.io repository for the content of our website and the main Bioschemas repository to create tasks as well as rising ideas, issues, opportunities. We do also use the [GitHub wiki](https://github.com/BioSchemas/bioschemas/wiki) section to keep few dynamic pages like meetings or projects.
